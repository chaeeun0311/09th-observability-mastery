## 2. 세션 목표

- 스터디 목적과 운영 방향 공유
- 사전 설문 결과 기반 커리큘럼 조정 방향 안내
- 과제 제출, 발표, 토론 방식 안내
- 후반부 실습 방향 안내
- 참여자별 기대사항과 배경 공유

---

## 3. 주요 안내 사항

- 스터디는 매주 목요일 21:00 ~ 22:30에 진행
- 매주 세션 전 `members/{영문이름}/weekXX.md` 형식으로 과제 제출
- 제출한 과제를 바탕으로 랜덤 발표자를 선정하고 발표 후 토론 진행
- 발표자를 미리 고정하지 않고 모두가 준비하는 흐름 유지
- 후반부 실습은 Base와 Project 두 방향으로 운영
- Base는 책 예제와 기본 데모 중심, Project는 개인 프로젝트나 졸업 프로젝트 적용 중심

---

## 4. 참여자 기대사항

| 관심사 | 내용 |
|---|---|
| 개념 정리 | Observability, Monitoring, Metrics, Logs, Traces를 설명 가능한 수준으로 이해 |
| 운영 관점 | 장애 상황에서 어떤 지표를 보고 판단할지 학습 |
| 도구 활용 | Prometheus, Grafana, OpenTelemetry, Kubernetes 기반 모니터링 흐름 이해 |
| 프로젝트 적용 | 개인 프로젝트, 졸업 프로젝트, 실무 환경에 Observability 적용 |
| 커리어 준비 | 면접이나 실무 대화에서 모니터링/운영 개념을 구체적으로 설명 |

---

## 5. 운영 관점 인사이트

- 도구를 설치하는 것보다 어떤 데이터를 보고 어떤 판단을 할지 정하는 것이 중요
- 책 요약만으로 끝내지 않고 운영 사례, Q&A, 토론을 함께 가져가는 방식 필요
- 실습 난이도와 참여자 배경이 다르므로 공통 Base와 개인 Project를 병행하는 구조 적합
- 과제 제출과 랜덤 발표는 참여 밀도를 유지하기 위한 장치
